4. Per-Stage Prompt Templates
INSTRUCTIONS

What it is: a structured prompt template for each stage that bundles the relevant slices of documents 1–3 with the specific task.
Why it prevents failures: AI-generated code quality degrades sharply when context is vague. "Write a PC layer" produces generic code. "Write a PC layer satisfying this interface contract, operating on tensors of this shape, in this environment, following this update equation" produces code that works on the first integration attempt.

INTERFACE CONTRACT
------------------
[FROM INTERFACE CONTRACT DOC §3 — PCLayer]

Class: PCLayer
__init__(self, in_features: int, out_features: int)
forward(self, x: Tensor) -> Tensor
  x: (B, in_features) float32
  returns: (B, out_features) float32  (initial representation r_init)

inference_step(self, r_below: Tensor, r: Tensor, e_above: Tensor, lr_infer: float) -> Tensor
  r_below: (B, in_features) float32
  r:       (B, out_features) float32
  e_above: (B, out_features) float32   (zero tensor at top level)
  lr_infer: float
  returns: (B, out_features) float32   updated r

update_weights(self, e: Tensor, r: Tensor, lr_weights: float) -> None
  e: (B, in_features) float32  final prediction error
  r: (B, out_features) float32 final representation
  lr_weights: float
  Updates self.W in-place (no gradients).

Invariants:
  - self.W shape (in_features, out_features), initialised as randn * 0.01.
  - No gradients on self.W during inference_step.
  - e_above is exactly self.W_next.T @ e_next, already projected by the caller.
  - precision weighting not used in this stage (uniform).
  - forward() must compute r_init = f(x) and may use a learned projection (e.g., x @ W_bottom_up) but that W_bottom_up is separate from self.W.

DATA FLOW CONTEXT
-----------------
The MNIST PC hierarchy is:
  x = mnist_image.view(B, 784)                 # (B, 784) float32
  r1 = pc1.forward(x)                          # (B, 256)
  e1 = x - pc1.W @ r1                          # (B, 784)  (top-down prediction of x)
  r2 = pc2.forward(r1)                         # (B, 64)
  e2 = r1 - pc2.W @ r2                         # (B, 256)
  r3 = pc3.forward(r2)                         # (B, 10)
  e3 = r2 - pc3.W @ r3                         # (B, 64)

Inference update (one step, η = lr_infer):
  # e_above for pc1 is pc2.W.T @ e2  (already computed by the hierarchy loop)
  r1 = pc1.inference_step(x,      r1, pc2.W.T @ e2, lr_infer)
  r2 = pc2.inference_step(r1,     r2, pc3.W.T @ e3, lr_infer)
  r3 = pc3.inference_step(r2,     r3, zeros(10),    lr_infer)   # top level

After convergence (fixed number of steps T):
  pc1.update_weights(e1, r1, lr_weights)
  pc2.update_weights(e2, r2, lr_weights)
  pc3.update_weights(e3, r3, lr_weights)

UPDATE EQUATION
---------------
From README §5.5:
  r_i  ←  r_i  +  η · ( e_i  −  W_{i+1}ᵀ · e_{i+1} )
where e_i = r_{i-1} − W_i @ r_i   (prediction error from layer below)
      e_{i+1} is the error from the layer above, back-projected.

In inference_step, this means:
  1. prediction = self.W @ r
  2. e = r_below - prediction
  3. delta_r = lr_infer * (e - self.W.T @ e_above)   # e_above already projected
  4. return r + delta_r

TASK
----
Implement class PCLayer in models/pc_layer.py satisfying the above contract exactly.

Requirements:
  - No gradients flow through self.W during inference_step (use torch.no_grad() or detach).
  - e_above is passed as a tensor of shape (B, out_features); do not compute W.T @ e_above inside inference_step — it is already projected by the caller.
  - forward() must set the initial r; simplest valid: r_init = x @ W_init, where W_init is a Parameter of shape (in_features, out_features), *separate from self.W*.
  - __repr__ must print in_features, out_features.
  - All dtypes float32.
  - Device-agnostic (use the device of input tensors).

DO NOT:
  - Add any functionality not specified (no batch norm, activation functions, logging, extra regularisation).
  - Use any library not listed in the environment.
  - Flatten or reshape spatial dimensions (this stage is already flat).
  - Allow gradient flow through self.W during inference_step.
  - Modify r_below or e_above in-place.

Class: JEPATokenizer
__init__(self, image_size=32, patch_size=4, dim=128, depth=4, heads=4, mlp_dim=256, predictor_depth=2, momentum=0.996)

encode(self, x: Tensor) -> Tensor
  x: (B, 3, 32, 32) float32  (normalised)
  returns: (B, 8, 8, 128) float32   2D token grid PRESERVED

encode_target(self, x: Tensor) -> Tensor
  x: (B, 3, 32, 32) float32
  returns: (B, 8, 8, 128) float32   detached, no grad

forward(self, x: Tensor, mask: Tensor) -> Tensor   (training only)
  x: (B, 3, 32, 32) float32
  mask: (B, 64) bool  (True = masked token positions)
  returns: (B, M, 128) float32  where M = mask.sum(dim=1) (varies per sample)

Function: jepa_loss(predicted: Tensor, target: Tensor) -> Tensor
  predicted: (B, M, 128) float32
  target:    (B, M, 128) float32 (detached)
  returns: scalar () float32 = 1 - mean(cosine_similarity)

Function: update_target_encoder(online_encoder, target_encoder, momentum=0.996) -> None
  Updates target_encoder parameters with EMA; target_encoder must have requires_grad=False permanently.

Invariants:
  - Context encoder (ViT) must apply 2D positional embeddings to the patch grid.
  - encode() is NOT used during training; only forward() with masking.
  - encode_target() runs under torch.no_grad() and returns a detached tensor.
  - When frozen, all parameters of context encoder have requires_grad=False.
  - The token grid output of encode() is (B,8,8,128), not flattened.

DATA FLOW CONTEXT
-----------------
Training step (simplified, inside autocast):
  x, mask = next(batch)                              # x: (B,3,32,32), mask: (B,64)
  # online encoder + predictor
  predicted_tokens = jepa.forward(x, mask)            # (B, M, 128)
  with torch.no_grad():
      target_tokens = jepa.encode_target(x)           # (B, 8,8,128) detached
      # gather targets at masked positions
      target_masked = target_tokens.reshape(B,64,128)[mask]  # (B, M, 128)
  loss = jepa_loss(predicted_tokens, target_masked)
  loss.backward()
  optimizer.step()
  update_target_encoder(jepa.context_encoder, jepa.target_encoder, momentum=0.996)

Token extraction for PC (after training, frozen):
  grid = jepa.encode(image)                           # (B,8,8,128)

---------------
Not directly applicable for JEPA training — but the loss is cosine similarity.

  e: (B, H, W, out_channels) float32

  Updates the convolution kernel(s) via local Hebbian rule.

Invariants:
  - self.W (the convolution kernel) shape (out_channels, in_channels, k, k). Must NOT require gradients during inference_step.
  - precision is already normalised (e.g., divided by sum over spatial positions) — do not renormalise.
  - The spatial layout H,W is preserved.
  - forward() initialises r using a simple convolution: conv(x, W_init) where W_init is a separate weight from self.W.

UPDATE EQUATION
---------------
From README §5.5:
  r_i  ←  r_i  +  η · ( e_i  −  W_{i+1}ᵀ · e_{i+1} )
For ConvPCLayer, the back‑projection W_{i+1}ᵀ · e_{i+1} is provided by the caller as e_above (already computed). So inference_step simply does:
  r_new = r + lr * (precision * e_below - precision * e_above)

TASK
----
Implement ConvPCLayer in models/pc_conv.py satisfying the above contract.

Requirements:
  - Store a convolution kernel `self.W` (the top‑down prediction weight) as a Parameter, but ensure it never gets gradients during inference_step (use torch.no_grad() context inside the method).
  - Include a separate initial projection weight `self.W_init` (Conv2d(in_ch, out_ch, k, padding=k//2)) used only in forward().
  - inference_step must NOT use self.W directly; it only needs the passed errors. The equation is r_new = r + lr_infer * (precision * e_below - precision * e_above). No convolutions inside inference_step — the caller already computed e_below and e_above in the correct shapes.
  - update_weights performs a local Hebbian update: dW = conv2d(r, e) aggregated over batch, then self.W.data += lr_weights * dW. Use torch.nn.functional.conv2d with appropriate padding and no gradient tracking.
  - __repr__ must show in_channels, out_channels, kernel_size.
  - All tensors float32, device consistent.

DO NOT:
  - Perform any convolution inside inference_step — it must be a simple element‑wise update using provided errors.
  - Renormalise precision — it arrives normalised.
  - Flatten spatial dimensions anywhere inside the layer.
  - Let self.W accumulate gradients during inference (detach or no_grad).
  - Add batch norm, dropout, or any regularisation not specified.

TASK
----
Implement JEPAPCComposite in models/composite.py and the two evaluation functions in experiments/evaluation.py exactly satisfying the contracts.

For JEPAPCComposite:
  - Store JEPA tokenizer (frozen, no grad).
  - Store pc_levels list.
  - Implement `_predict_down(level, r)` for each level: conv for levels 1&2, fc for level3.
  - Implement `_upsample_error` and `_project_above` to prepare e_above for each level. Must ensure shapes match the contracts (e.g., conv_transpose to match channels, and spatial upsampling when moving up).

  - All dtypes float32. Use torch.no_grad() during the inference loop.

For evaluation functions:
  - `evaluate_inference_adaptation`: apply occlusion function to images, run `model.inference()` with varying steps, train a linear probe on the final representation of a chosen level (e.g., r3), return accuracy dict.
  - `goal_directed_inference`: given tokens, run inference loop manually, compute mean uncertainty as (1/precision).mean(), break if below threshold. Return final r_list, prec_list, steps_taken.

Requirements:
  - No gradient flows to JEPA or PC weights during inference.
  - Precision normalisation must be within-level (e.g., divide by sum over spatial positions of precision tensor for that error).
  - The __repr__ of JEPAPCComposite must show the architecture shapes (token grid dims, PC level shapes).
  - Include the simple linear probe inside the evaluation function (using sklearn or a torch Linear layer trained with LBFGS/Adam on frozen features).

  - Fine‑tune the JEPA tokenizer during PC training or inference.
  - Renormalise precision inside ConvPCLayer or PCLayer — it is done once in the composite.
  - Use any non‑local weight updates; PC weights must use local Hebbian (or no‑grad) as specified.
  - Add extra logging, TensorBoard, or checkpointing inside the core modules.
  - Flatten or discard the 2D token grid at any point.
  - Implement any goal‑directed logic inside the model itself; that belongs in the evaluation harness.
