2. Interface Contract Document (the most important one)
INSTRUCTIONS:

What it is: for every class and function that will be written, specifies the exact input shapes, output shapes, dtypes, and any invariants — before any implementation is written.
Why it prevents failures: the most common reason AI-generated modules fail on integration is shape mismatches at boundaries. The JEPA tokenizer outputs (B, 64, 128). PC Level 1 expects (B, 8, 8, 128). That reshape needs to be in the contract, not discovered at runtime.
What it must contain, for every module:

def update_target_encoder(online_encoder: nn.Module, target_encoder: nn.Module, momentum: float = 0.996) -> None:
    """
    Exponential moving average update of the target encoder's parameters.
    Target encoder NEVER receives a gradient — this is enforced outside this function.
    """

    Parameters
    online_encoder – nn.Module, source of the latest weights (trained by gradient descent)
    target_encoder – nn.Module, recipient of the EMA weights
    momentum – float, decay factor for EMA (must be in [0, 1])

    Returns
    None — updates target_encoder parameters in-place.

    Invariants

        Must be called after every optimizer step on online_encoder.

        target_encoder parameters must have requires_grad = False at all times (set once after creation).

        The update rule for every parameter p_online, p_target:
        p_target.data = momentum * p_target.data + (1 - momentum) * p_online.data

        No gradients are ever computed for the target encoder.

def encode(self, x: Tensor) -> Tensor:
    """Extract token grid from an image using the online context encoder.
       This is the entry point for the PC hierarchy.
       NEVER returns gradients to the JEPA when the tokenizer is frozen."""

    Input
    x – (B, 3, image_size, image_size), float32, pixel values in [0, 1] or normalized.

    Output
    Token grid: (B, 8, 8, dim) = (B, 8, 8, 128), float32.
    Spatial layout preserved — token at [h, w] corresponds to the image patch at that position.
    Positional embeddings are included inside the encoder.

    Invariants

        When the tokenizer is frozen, all parameters of the context encoder have requires_grad = False.

        The output retains the 2D grid shape; it is never flattened.

        Two calls with the same input must return identical tokens (deterministic).

        The encoder runs in eval() mode during PC training/inference.

    Input

    Output

    Invariants

        This method is called exactly once per input before starting the inference loop.

Method: inference_step
python

def inference_step(self, r_below: Tensor, r: Tensor, e_above: Tensor, lr_infer: float) -> Tensor:
    """Perform one step of the PC inference update for this layer.
       Weights are NOT updated — they are held fixed.
       `r` is updated in-place or a new tensor is returned (contract: returns updated r)."""

    Parameters
    r_below – (B, in_features), float32, representation of the layer below.
    r – (B, out_features), float32, current representation of this layer.
    e_above – (B, out_features), float32, prediction error from the layer above this one (already projected down via W_next.T by the layer above). If this is the top level, e_above is a zero tensor.
    lr_infer – float, inference step size (η).

    Returns
    r_updated – (B, out_features), float32, updated representation after one inference step.

    Computation

        prediction = self.W @ r # top-down prediction of r_below

        e = r_below - prediction # prediction error from below (this level's error)

        delta_r = lr_infer * (e - self.W.T @ e_above) # energy gradient

        r_new = r + delta_r

        Return r_new

    Invariants

        self.W (shape (in_features, out_features)) must not receive gradients during inference_step. No W.grad is populated; weights are held constant.

        The computation must not modify r_below or e_above.

        If e_above is not provided (top level), the term self.W.T @ e_above is omitted (equivalent to e_above = zeros).

        Precision weighting is not applied in Stage 1; this method assumes uniform precision. (Later, precision can be integrated as element-wise multiplication of e and e_above.)

Method: update_weights
python

def update_weights(self, e: Tensor, r: Tensor, lr_weights: float) -> None:
    """Hebbian weight update: W += lr_weights * outer(e, r). Called only after inference has converged."""

    Parameters
    e – (B, in_features), float32, final prediction error from below (after convergence).
    r – (B, out_features), float32, final representation of this layer.
    lr_weights – float, learning rate for weights.

    Returns
    None — updates self.W in-place.

    Computation
    self.W += lr_weights * torch.einsum('bi,bo->io', e, r) / B (or using batch outer product sum).

    Invariants

        Must be called only after the inference loop has converged (errors are stable).

        No gradient book-keeping (pure Hebbian, local).

        e and r are detached from the inference graph.

Method: inference_step
python

def inference_step(self,
                   r: Tensor,
                   r_above: Tensor,
                   e_below: Tensor,
                   e_above: Tensor,
                   precision: Tensor) -> Tensor:
    """One inference update for this convolutional PC layer."""

    Parameters
    r – (B, H, W, out_channels), float32, current representation at this level.
    r_above – (B, H', W', out_channels_above), float32, representation from the level above (spatially coarser if pooling is used between levels). This is used to compute the top-down prediction to this level.
    e_below – (B, H, W, out_channels), float32, prediction error from the level below (the mismatch between r and what the level below expected). In this convolutional setting, the error is defined at the same spatial positions as r.
    e_above – (B, H, W, out_channels), float32, prediction error from the level above — that is, the error from the next level up, already projected/upsampled to match the spatial and channel dimensions of this level. The caller (PC hierarchy) is responsible for ensuring the projection.
    precision – (B, H, W, out_channels), float32, precision weights (inverse variance) at each spatial location and channel. Must be normalised within the level before this call (see invariant). In Stage 1–3 vanilla versions, precision may be a tensor of ones.

    Returns
    r_updated – (B, H, W, out_channels), float32, updated representation.

    Computation outline

        e = e_below (or e_below - W_conv(r_above) if using explicit top-down prediction; here e_below already contains the error from the lower layer).

        The update rule:
        r_new = r + lr_infer * (precision * e - W_back.T * (precision * e_above))
        where W_back.T is a transposed convolution that maps e_above (which has same channels as r) back to the same shape but through the receptive field. In a simple implementation using standard convolution, the top-down weight W is the convolution kernel mapping r to a prediction of the layer below. Thus e_below is already the prediction error. The term from above is the back-projection of the error from above through W^T.

        The inference learning rate lr_infer is a global hyperparameter or per-layer.

    Invariants

        No gradients on weights: The convolution kernels self.W (used to generate predictions) must not accumulate gradients during inference_step.

        Precision normalisation: precision must be normalised within the level prior to this call (e.g., dividing by the sum over spatial positions or per-channel, to avoid scale differences across levels).

        Spatial layout: The output r_updated must preserve the same spatial grid (H, W) as the input r. The spatial mapping between levels (e.g., upsampling of e_above to this level's grid) is the responsibility of the hierarchy, not this layer.

        The update does not modify r in-place; it returns a new tensor. The caller overwrites r.

Method: update_weights
python

def update_weights(self, e: Tensor, r: Tensor, lr_weights: float) -> None:
    """Local Hebbian update for the convolutional kernel."""

    Parameters
    e – (B, H, W, out_channels), float32, final prediction error (from below).
    r – (B, H, W, out_channels), float32, final representation at this level.
    lr_weights – float, learning rate.

    Returns
    None — updates self.W in-place.

    Computation
    Approximate outer product over patches: for each local patch of r and e, accumulate the outer product to the kernel. Implementation detail: can be done via torch.nn.functional.conv2d with gradient disabled. Exact contract: the weight update is proportional to the sum over all positions of the outer product of the error vector at a position and the representation vector at the corresponding position in the receptive field.

    Invariants

        Called only after inference has converged.

        No gradient flow through this update.

    """Run the full PC inference loop on an image. Returns final representations, prediction errors, and precision tensors for each level."""

    Input

    Output

        Precision tensors are normalised within each level before being used in the inference_step calls.

        The tokenizer is run once at the beginning; tokens are reused across inference steps.

7. Common Invariants Across All Modules

    Dtype: All tensors are torch.float32 unless stated.

    Device: All tensors within a forward pass must reside on the same device (CPU or GPU).

    Gradient flow: During inference (PC loop), gradients must not be tracked. Use torch.no_grad() context or detach appropriately. During weight updates, only the relevant weights receive manual updates (no .backward() through the inference graph).

    Batch consistency: All functions assume the first dimension is batch; no single-sample branching that breaks batched operation.

    Reproducibility: All random initialisations must be seedable. The EMA and inference loop are deterministic given fixed inputs and seeds.

This contract is the source of truth. Any code that violates these shapes, dtypes, or invariants is incorrect, no matter how plausible it looks.
